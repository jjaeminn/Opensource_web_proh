AuthPage.tsx에서 
로그인/회원가입 두 가지 모드를 하나의 페이지에서 처리함
isLogin state로 true면 로그인 폼, false면 회원가입 폼 보여줌

state 구성 : 이름, 이메일, 비밀번호, 에러메시지, 비밀번호 표시여부, 로그인/회원가입 모드 구분

handleSubmit() :
isLogin 여부에 따라 분기
로그인이면 → POST /auth/login 으로 email, password 보냄
회원가입이면 → POST /auth/register 로 name, email, password 보냄
서버 응답에서 success가 false면 errMsg state에 메시지 저장 → 버튼 위에 빨간 글씨로 표시

auth.js (백엔드)
POST /auth/register : 회원가입

프론트에서 name, email, password, adminCode 받음
필수 항목 빠지면 바로 실패 응답 리턴
이미 있는 이메일이면 중복 응답 리턴
bcrypt로 비밀번호 암호화 후 DB 저장
adminCode가 'ADMIN1234' 이면 role : admin, 아니면 role : user 로 저장
저장 완료되면 success: true + 유저 정보 리턴
POST /auth/login : 로그인

프론트에서 email, password 받음
이메일로 DB에서 유저 찾기 → 없으면 실패 응답
bcrypt.compare로 암호화된 비밀번호랑 입력값 비교 → 틀리면 실패 응답
통과되면 success: true + id, name, email, role 리턴
ㄴ role 값을 프론트에서 받아두면 나중에 관리자/일반 페이지 분기할 때 활용 가능
error msg는 console.log으로 바꿈
